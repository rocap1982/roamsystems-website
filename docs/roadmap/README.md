# Roadmap

This directory contains the product roadmap and milestone tracking.

## Files

- `ROADMAP.md` — the product roadmap with phases, priorities, and status.

## How it works

- The roadmap organizes work into **phases** (sequential or parallel tracks).
- Each phase has a status: `not started`, `in progress`, `completed`.
- Sprints link to roadmap phases to show which work advances which goal.
- `/close-session` and `/review-sprint` check whether completed work should update roadmap status.

## Maintenance

- Update `ROADMAP.md` when a sprint completes a phase or milestone.
- Keep phases high-level — detailed task breakdowns belong in sprint docs.
- Reference specific plan/sprint docs from phase descriptions for traceability.
