# Documentation Hierarchy (Canonical)

This document defines the documentation hierarchy and source-of-truth precedence for this repository.

## Canonical source-of-truth stack

1. Documentation standards and this hierarchy
2. Canonical schemas and contracts
3. Overview documents
4. API contract documents
5. Database documentation
6. UI documentation
7. Glossary and cross-module reference

## Enforcement rules

- Higher-precedence documents govern conflicts.
- Lower-precedence documents do not introduce, rename, or redefine fields owned by higher-precedence schemas.
- Cross references point to authoritative documents and do not duplicate schema definitions.
